## Objectif
Créer une interface de **lettrage manuel** accessible depuis le menu principal, permettant à l'utilisateur de sélectionner deux lignes d'écriture (une débit, une crédit) du même compte de tiers, de spécifier un montant à lettrer, et d'enregistrer le lettrage via l'action serveur existante `createLettrage`.

Cette interface s'inspire du workflow d'Odoo : l'utilisateur voit les écritures ouvertes (résiduel > 0), choisit une ligne débit et une ligne crédit du même compte, saisit le montant, et valide.

### 2. Server actions à ajouter / modifier
Dans `src/server/lettrage.ts` (déjà présent) :
- **`export async function getOpenLines(dossierId: string): Promise<OpenLineDTO[]>`**  
  Retourne toutes les lignes d'écriture non totalement lettrées (`amountResidual > 0`) appartenant à des pièces non annulées, avec les informations suivantes :
  - `id` (ligne)
  - `pieceId`
  - `pieceNumero` (numeroPiece)
  - `pieceDate` (datePiece en ISO string)
  - `journalCode`
  - `compteNumero`
  - `intituleCompte` (libellé du compte)
  - `libelleLigne`
  - `debit` (number)
  - `credit` (number)
  - `amountResidual` (number, absolu)
  - `sens` : 1 si ligne débit (debit > credit), -1 si ligne crédit (credit > debit)  
  - `pieceStatut` (BROUILLON/VALIDEE/ANNULEE)  

  La fonction doit éviter de retourner les lignes des pièces ANNULEE.

  Format DTO :
  ```typescript
  export interface OpenLineDTO {
    id: string;
    pieceId: string;
    pieceNumero: string;
    pieceDate: string; // ISO
    journalCode: string;
    compteNumero: string;
    intituleCompte: string;
    libelleLigne: string;
    debit: number;
    credit: number;
    amountResidual: number; // absolu (|débit - crédit|)
    sens: 1 | -1; // 1 = débit, -1 = crédit
    pieceStatut: 'BROUILLON' | 'VALIDEE' | 'ANNULEE';
  }
  ```

### 3. Composant client `LettrageClient.tsx`
- Deux panneaux latéraux (gauche/droite) similaires à `EcrituresClient` mais adaptés :
  - **Panneau gauche** : liste des lignes ouvertes, filtrable par compte ou par pièce, avec possibilité de sélectionner une ligne (débit ou crédit).
  - **Panneau droit** : détails de la ligne sélectionnée, et formulaire de lettrage quand deux lignes sont sélectionnées (une débit, une crédit, même compte).
- État client :
  - `selectedDebitLineId: string | null`
  - `selectedCreditLineId: string | null`
  - Quand les deux sont sélectionnés et appartiennent au même compte, afficher :
    - Montant maximum lettrable = min(amountResidual des deux lignes)
    - Champ de saisie nombre « Montant à lettrer » (avec validation : > 0 et ≤ montant max)
    - Bouton « Lettrer » (appelle `createLettrage` avec `auto: false`)
    - Bouton « Réinitialiser »
- Lorsqu'un lettrage est créé avec succès :
  - Rafraîchir la liste des lignes ouvertes (via `getOpenLines`)
  - Réinitialiser les sélections
  - Afficher une notification succincte (ex: alert ou toast simple) – on peut utiliser `window.alert` pour simplicité dans le POC.
- Gestion des erreurs : afficher l'erreur renvoyée par l'action serveur.
